# 產品宏觀規劃：整合資產管理平台（Wealth Dashboard）

## 1. 市場痛點

- **資訊分散**：個人的銀行存款、證券戶、加密貨幣、NFT 等資產散落在不同平台，難以一目了然
- **缺乏整合視角**：沒有單一工具能同時看到淨資產的全貌
- **手動更新繁瑣**：許多工具需要手動輸入或匯入資料
- **決策困難**：沒有統一的資產配置視覺化，難以評估風險與報酬

## 2. 目標用戶

- **主要**：30–55 歲的理財愛好者、有多帳戶（銀行、股票、加密）的個人投資者
- **次要**：理財顧問、家族辦公室（Family Office）
- **潛力**：追求財務自由的 FIRE 族群、準備退休規劃的中年人

## 3. 市場定位（差異化）

| 競品 | 缺點 | 我們的切入點 |
|------|------|-------------|
| Mint | 僅支援美國銀行，無加密 | 支援台灣銀行 + 加密交易所 |
| Personal Capital | 介面複雜，收費制 | 免費 MVP + 直覺儀表板 |
| Money Hub（日本） | 無中文、無 API 整合 | 在地化 + 開放 API |

**核心定位**：亞洲（優先台灣）一站式「個人資產總覽」平台，主打「一個儀表板，看懂你的全部身家」。

## 4. 功能規格

### MVP（P0）

| 功能 | 描述 |
|-----|------|
| 帳號連結（Read-only） | 透過 Open Banking / API 串接銀行、證券、加密交易所（讀取餘額） |
| 淨資產儀表板 | 單一頁面顯示總資產、總負債、淨資產 |
| 資產配置圖 | 圓餅圖呈現各類別（股票、保險、加密、現金）比例 |
| 帳戶總覽列表 | 各帳戶餘額、昨日/週/月變化 |
| 幣別整合 | 自動匯率換算，顯示單一幣別（USD/TWD）總資產 |

### P1 功能

| 功能 | 描述 |
|-----|------|
| 歷史趨勢圖 | 淨資產隨時間變化（Line Chart） |
| 收益分析 | 各投資標的之報酬率（% 和絕對值） |
| 警示通知 | 資產大幅下跌/上涨提醒（Push / Email） |
| 負債管理 | 信用卡/貸款利率追蹤與提醒 |

### P2 功能

| 功能 | 描述 |
|-----|------|
| AI 財務建議 | 根據資產配置給予調整建議 |
| 預算追蹤 | 月支出 vs 收入分析 |
| 稅務報告摘要 | 年度資本利得/股息總結 |
| 多成員共享 | 家庭成員資產視圖 |

## 5. 技術建議（提供給 Alan）

### 安全性（最高優先）
- 所有金融資料僅做 **Read-only** 串接，嚴禁任何寫入/交易操作
- Token 儲存使用 **AES-256 加密**，不儲存明文
- 符合 **PSD2 / Open Banking** 規範，或透過合法 aggregator（如 Plaid, Yodlee）串接

### 資料來源整合方向
- **銀行**：台灣可透過財金公司 API 或 Open Banking 第三方 TSP
- **證券**：券商 API（如元大、富果）讀取持股與餘額
- **加密**：交易所 API（幣安、Coinbase）Read-only Key
- **參考架構**：`FinTech Aggregator` 模式，類似於美國的 Plaid

### 前端建議
- Dashboard 使用 **React + D3.js / Recharts** 绘制图表
- Mobile-first，響應式設計

### 技術棧建議
```
前端: Next.js + Tailwind CSS + Recharts
後端: Node.js / Fastify
資料庫: PostgreSQL（資產資料）+ Redis（Session Cache）
認證: JWT + OAuth 2.0
部署: Vercel（前端）+ Railway/Render（後端）
```

## 6. 成功指標（KPI）

| 指標 | 目標 |
|------|------|
| MAU（每月活躍用戶） | 上線 3 個月內 1,000 人 |
| 連結帳戶數 | 人均 3 個帳戶以上 |
| 留存率 | 30 日留存 ≥ 40% |
| 淨資產數據完整率 | ≥ 90% 用戶完成至少 1 個帳戶連結 |
| 用戶滿意度（NPS） | ≥ 50 |

## 7. 上線標準（MVP）

- [ ] 可連結至少 1 家台灣銀行帳戶（Read-only）
- [ ] 儀表板顯示正確淨資產數字
- [ ] 支援加密貨幣帳戶（至少 Binance Read-only）

### UI/UX
- [ ] 首頁載入 < 2 秒
- [ ] 響應式設計（Mobile + Desktop）
- [ ] 無資料時有引導填空狀態

### 安全
- [ ] 所有 API 為 Read-only
- [ ] 無敏感資料明文儲存
- [ ] OAuth 登入（Google / Apple）
